Documentation

Here are listed all the classes and methods from the engine.

Maned

Maned is the main module of the engine, where are contained the game methods itself and other submodules.

Creating a game

Constructor object

className Class name of the Canvas element that the engine will be atached
width the screen width of the game
height the screen height of the game
background the color of the background. Must be a hex color.
fps frame rate of the game (frames per second)

Methods

init init the instanciated game
on receives the event of the game and a callback

Events

preload event that runs before the game runs. It's used to preload the nedeed assets
start runs in the first frame before the game loop begins
update runs in the game loop itself. The code in this event will run every frame of the game
pause runs code when the game is paused
menu runs code to create a menu at the beginning of the game
credits runs code at the end of the game

Example of an implementation of a event

Controls

Controls is the module that manages the keyboard mapping and key pressed events.

Method

isPressedKey detects if a key is pressed. Receives a mapped key string

Preloader

Receives an array of assets (images, json or sounds) and preloads then.

Method

preload receives an array of strings with the paths of the required assets
images access the preloaded images in this array by string index

Sprite

One of the most important modules of the engine. You can use this class to instanciate any interactable objects in the game.

Constructor object

image preloaded asset object
x x position of the canvas where the object will be created
y y position of the canvas where the object will be created
width width of the sprite
height height of the sprite
delay delay in fps of the animation
frames number of frames of the animation
loop sets if the animation has a loop or not
animated sets if the sprite is animated or not

Methods

render render the sprite in the canvas
update update the animation each frame

Collision

Module that detects sprite collision.

Method

isCollided checks a collision between two sprite objects

Tile

Module that manages a tile and renders in the canvas.

Consructor parameters

json object json object that contains the tile data. The json can be preloaded in the preloader module
image object image object that contains the tileset
